# 19 — Credit Trust & Affordability Invariant

Функции автокредита должны повышать прозрачность и предсказуемость для пользователя.
Кредитные расчёты рассматриваются как часть доверия к маркетплейсу, а не только как вспомогательная функция.

---

## Core Rules

### 1. Calculation Transparency

Любой отображаемый ежемесячный платёж должен быть объясним пользователю.

Backend обязан возвращать структуру расчёта:

```
price
downPaymentAmount
loanPrincipal
interestRateUsed
monthsUsed
monthlyPayment
```

UI только отображает данные и не выполняет финансовых вычислений.

---

### 2. Server Calculation SSOT

Все кредитные расчёты выполняются исключительно на сервере.

Frontend никогда не содержит формул кредита или альтернативных вычислений.

---

### 3. Affordability Snapshot

При создании или обновлении объявления система должна сохранять вычисленное значение:

```
estimatedMonthlyFrom
```

Это значение используется для:

- списка объявлений
- карточек превью
- результатов поиска

Расчёты не выполняются в списках объявлений в реальном времени.

---

### 4. Non-Blocking Trust Principle

Кредитные данные:

- не влияют на публикацию объявления
- не изменяют ranking
- не изменяют lifecycle listing

Они являются исключительно информационным слоем доверия.

---

### 5. Neutral Messaging

Все кредитные значения должны сопровождаться нейтральным дисклеймером:

> «Расчёт ориентировочный и не является банковским предложением»

Система не должна создавать ложного ощущения гарантированного одобрения кредита.

---

## Goal

Повысить доверие пользователей через прозрачные финансовые расчёты и обеспечить масштабируемость кредитной функциональности без нарушения архитектуры маркетплейса.
